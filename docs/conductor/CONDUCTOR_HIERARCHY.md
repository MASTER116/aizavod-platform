# CONDUCTOR -- Иерархия управления Zavod-ii

## Версия 1.0 | 2026-03-16

---

## Принцип

CONDUCTOR принимает задачу от основателя и декомпозирует её по трёхуровневой иерархии:

```
ОСНОВАТЕЛЬ (идея/задача)
    ↓
CONDUCTOR (мета-оркестратор)
    ↓ декомпозиция по директорам
ДИРЕКТОРА (C-level)
    ↓ декомпозиция по отделам
НАЧАЛЬНИКИ ОТДЕЛОВ
    ↓ декомпозиция по специалистам
СПЕЦИАЛИСТЫ (исполнители)
    ↓
РЕЗУЛЬТАТ → сборка → отчёт основателю
```

## Уровень 3: СПЕЦИАЛИСТЫ

Специалисты — конечные исполнители внутри отделов. Примеры:

- `cto.backend.fastapi_dev` — разработчик FastAPI
- `cto.backend.db_admin` — администратор БД
- `cto.devops.docker_engineer` — Docker/инфраструктура
- `cto.ai.prompt_engineer` — промпт-инженер
- `cfo.grants.rnf_writer` — составитель заявок РНФ
- `cmo.content.instagram_creator` — контент-мейкер Instagram
- `cdo.uiux.web_designer` — веб-дизайнер
- `clo.ip.trademark_analyst` — аналитик товарных знаков

Специалисты создаются динамически по мере необходимости. Каждый имеет:
- Системный промпт (роль, компетенции, ограничения)
- Доступ к релевантным сервисам и данным
- Отчётность перед начальником отдела

## Алгоритм работы CONDUCTOR

### Шаг 1: Приём задачи
Основатель отправляет задачу в свободной форме.

### Шаг 2: CEO-декомпозиция
CONDUCTOR (как CEO) анализирует задачу и определяет:
- Какие директора нужны
- Какой приоритет для каждого
- Какие зависимости между задачами

Результат: ConductorTask (parent) + подзадачи для директоров.

### Шаг 3: Директорская декомпозиция
Каждый директор получает свою подзадачу и разбивает на задачи для отделов.

Результат: подзадачи для начальников отделов.

### Шаг 4: Отдельская декомпозиция
Начальник отдела разбивает на конкретные задачи для специалистов.

Результат: конкретные исполнительные задачи.

### Шаг 5: Исполнение
Специалисты выполняют задачи (Claude Code = исполнитель всех ролей).

### Шаг 6: Сборка результатов
Результаты собираются снизу вверх:
Специалист → Начальник отдела → Директор → CONDUCTOR → Основатель.

## Пример декомпозиции

**Задача:** "Сделать лендинг Zavod-ii"

```
CONDUCTOR (CEO):
├── CTO: Разработать лендинг
│   ├── cto.frontend: Вёрстка React + Tailwind
│   │   └── web_developer: Код компонентов
│   ├── cto.backend: API для форм обратной связи
│   │   └── fastapi_dev: Эндпоинт /api/contact
│   └── cto.devops: Деплой на Nginx
│       └── docker_engineer: Обновить nginx.conf
├── CDO: Дизайн лендинга
│   ├── cdo.uiux: Макет и прототип
│   │   └── web_designer: Figma → код
│   └── cdo.brand: Фирменный стиль
│       └── brand_designer: Логотип, цвета, шрифты
├── CMO: Контент для лендинга
│   └── cmo.content: Тексты, CTA, SEO
│       └── copywriter: Написать тексты
└── CLO: Юридические страницы
    └── clo.contracts: Политика конфиденциальности
        └── legal_writer: privacy.html, terms.html
```

## Правила

1. **Не все директора нужны для каждой задачи.** Простые задачи могут идти напрямую к одному директору.
2. **Декомпозиция останавливается когда задача достаточно конкретна** для исполнения (< 2 часов работы).
3. **Зависимости:** если задача CDO блокирует CTO (нужен дизайн перед вёрсткой) — CONDUCTOR это учитывает.
4. **Приоритет:** Класс A (деньги) всегда выше Класса B (репутация).
5. **Каждый уровень отчитывается наверх** с результатом и статусом.
